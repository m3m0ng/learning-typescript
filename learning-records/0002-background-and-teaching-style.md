# Background: mechanical engineer, knows Python, prefers application-based teaching

The user is a mechanical engineer (not a software/programming background) with hands-on Python and Arduino experience from personal robotics projects. They have no JavaScript knowledge. Comparing TypeScript to JavaScript was confusing and added an unnecessary second unknown language.

## Implications for future lessons
- Teach TypeScript on its own terms, anchored to **Python** (a language they already know) for analogies — not JavaScript.
- Use **real-world, application-based framing** drawn from engineering/robotics contexts (sensors, motors, datasheets, control systems) rather than abstract programming examples.
- Avoid "here's JS, now here's TS" comparisons going forward.
- Keep pace deliberate — confirm understanding before layering new syntax.
