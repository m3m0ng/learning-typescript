# Notes

- User is a mechanical engineer (not a programmer) with hands-on Python and Arduino experience from personal robotics projects. No JavaScript knowledge.
- Primary motivation: be able to read and understand TypeScript produced by AI tools (function signatures, interfaces, generics, utility types) in real software — CLI tools, TUI components, web apps. Robotics is a comfort-language analogy only, NOT the example domain (see learning-records/0003).
- Teaching style: anchor explanations to Python (known language) for syntax intuition. Examples should be drawn from general software/CLI/TUI/web contexts, not robotics.
- JS vs TS framing: Lesson 1's "side-by-side JS then TS" framing (treating JS as a second language to learn first) was confusing — avoid that. Instead, from Lesson 3 onward, use small inline tags marking whether a given piece of syntax is plain JavaScript (runs at runtime) or TypeScript-only (compile-time types, vanish on build). TS-primary, JS noted in passing.
- Keep lessons short, beautiful, and tied to a single concept with a tangible win.
- Pace: go slower, confirm understanding before adding new syntax.
